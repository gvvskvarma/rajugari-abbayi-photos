# Week 1 Day 3 API Contract (Upload Pipeline)

Day 3 adds a reliable browser-to-R2 upload workflow with signed URLs and upload session tracking.

## Auth model

- Identity: Supabase Auth session JWT (`Authorization: Bearer <token>`)
- Upload URL issuing: admin-only and delivery ownership-checked
- Upload completion: requires valid `uploadToken` generated by request step

## Endpoints

### `POST /api/v1/request-upload-url`

Creates a short-lived upload session and returns a direct browser `PUT` URL to private R2.

Request body:

```json
{
  "deliveryId": "uuid",
  "fileName": "IMG_0042.CR3",
  "contentType": "image/x-canon-cr3",
  "fileSize": 45218812
}
```

Response body:

```json
{
  "objectKey": "deliveries/<deliveryId>/raw/<timestamp>-IMG_0042.CR3",
  "uploadToken": "hex-token",
  "uploadUrl": "https://<account>.r2.cloudflarestorage.com/<bucket>/...",
  "expiresInSeconds": 900,
  "maxFileBytes": 5368709120
}
```

Notes:
- URL expiry is 15 minutes.
- Files larger than 5GB are rejected.

### `POST /api/v1/upload/complete`

Finalizes upload metadata after browser upload succeeds.

Request body:

```json
{
  "deliveryId": "uuid",
  "objectKey": "deliveries/<deliveryId>/raw/<timestamp>-IMG_0042.CR3",
  "uploadToken": "hex-token",
  "fileName": "IMG_0042.CR3",
  "mimeType": "image/x-canon-cr3",
  "bytes": 45218812
}
```

Response body:

```json
{
  "ok": true,
  "assetId": "uuid",
  "uploadSessionId": "uuid"
}
```

Validation rules:
- `uploadToken` must match a non-expired upload session.
- `bytes` must be within tolerance of the requested size.
- Session can be finalized only once.

## Persistence changes

- New migration: `supabase/migrations/20260311_day3_upload_pipeline.sql`
- New table: `public.upload_sessions`
  - tracks request token, expected bytes, object key, status, expiry, attempts
  - uses RLS with owner/admin access

## Frontend upload reliability behavior

The Day 3 admin UI uses:
- request-upload-url API per file
- direct browser `PUT` to R2 signed URL
- retry up to 3 attempts with backoff for transient failures
- finalize metadata only after successful upload
